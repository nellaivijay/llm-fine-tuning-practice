## Data Cleaning and Validation

### Common Data Issues

**1. Duplicates**
```python
import pandas as pd

# Remove exact duplicates
df = df.drop_duplicates()

# Remove near-duplicates (using text similarity)
from difflib import SequenceMatcher

def is_duplicate(text1, text2, threshold=0.9):
    return SequenceMatcher(None, text1, text2).ratio() > threshold
```

**2. Missing Values**
```python
# Handle missing values
df = df.dropna()  # Remove rows with missing values
df = df.fillna("")  # Fill with empty string
df = df.fillna(df.mode().iloc[0])  # Fill with mode
```

**3. Inconsistent Formatting**
```python
# Standardize text
df['text'] = df['text'].str.lower().str.strip()
df['text'] = df['text'].str.replace(r'\s+', ' ', regex=True)
```

**4. Noise and Errors**
```python
# Remove special characters (if not needed)
import re
df['text'] = df['text'].apply(lambda x: re.sub(r'[^\w\s]', '', x))

# Fix encoding issues
df['text'] = df['text'].apply(lambda x: x.encode('utf-8', errors='ignore').decode('utf-8'))
```

### Data Validation

**Schema Validation**:
```python
# Check required fields
required_fields = ['input', 'output']
for field in required_fields:
    assert field in df.columns, f"Missing field: {field}"

# Check data types
assert df['input'].dtype == 'object', "Input should be text"
assert df['output'].dtype == 'object', "Output should be text"
```

**Quality Checks**:
```python
# Check text length
df['input_length'] = df['input'].str.len()
df = df[df['input_length'] > 10]  # Remove too short
df = df[df['input_length'] < 10000]  # Remove too long

# Check for empty strings
df = df[df['input'].str.strip() != '']
```

## Synthetic Data Generation

### When to Use Synthetic Data

**Benefits**:
- Augment limited datasets
- Cover edge cases
- Generate diverse examples
- Reduce data collection costs

**Use Cases**:
- When real data is scarce
- When privacy is a concern
- When specific scenarios are needed
- For testing and validation

### Generation Methods

**1. Template-Based Generation**
```python
templates = [
    "What is {topic}?",
    "Explain {topic} in simple terms.",
    "How does {topic} work?",
    "Why is {topic} important?",
]

topics = ["machine learning", "neural networks", "deep learning"]

synthetic_data = []
for template in templates:
    for topic in topics:
        synthetic_data.append(template.format(topic=topic))
```

**2. LLM-Based Generation**
```python
from openai import OpenAI

client = OpenAI(api_key="your-api-key")

def generate_synthetic_data(prompt, n=5):
    response = client.chat.completions.create(
        model="gpt-4",
        messages=[
            {"role": "system", "content": "Generate diverse training examples."},
            {"role": "user", "content": prompt}
        ],
        n=n
    )
    return [choice.message.content for choice in response.choices]

examples = generate_synthetic_data(
    "Generate 5 question-answer pairs about Python programming."
)
```

**3. Data Augmentation**
```python
# Paraphrasing
from transformers import pipeline

paraphraser = pipeline("text2text-generation", model="t5-base")

def augment_text(text):
    paraphrase = paraphraser(f"paraphrase: {text}", max_length=100)
    return paraphrase[0]['generated_text']
```

**4. Back-Translation**
```python
# Translate to another language and back
from transformers import pipeline

translator = pipeline("translation", model="Helsinki-NLP/opus-mt-en-de")
translator_back = pipeline("translation", model="Helsinki-NLP/opus-mt-de-en")

def back_translate(text):
    german = translator(text)[0]['translation_text']
    english = translator_back(german)[0]['translation_text']
    return english
```

## Data Formatting

### Common Data Formats

**1. Instruction Tuning Format**
```json
{
  "instruction": "Explain the concept of overfitting in machine learning.",
  "input": "",
  "output": "Overfitting occurs when a model learns the training data too well..."
}
```

**2. Conversation Format**
```json
{
  "messages": [
    {"role": "user", "content": "What is machine learning?"},
    {"role": "assistant", "content": "Machine learning is..."},
    {"role": "user", "content": "Can you give an example?"},
    {"role": "assistant", "content": "Sure! An example is..."}
  ]
}
```

**3. Completion Format**
```json
{
  "prompt": "The capital of France is",
  "completion": "Paris"
}
```

**4. QA Format**
```json
{
  "context": "Machine learning is a subset of artificial intelligence...",
  "question": "What is machine learning?",
  "answer": "A subset of artificial intelligence"
}
```

## Tokenization and Encoding

### Understanding Tokenization

**What is Tokenization?**
- Converting text to numerical representations
- Model-specific tokenizers
- Different tokenization strategies

**Tokenization Types**:
- Word-level: Split by words
- Subword-level: Split into meaningful subwords (BPE, WordPiece)
- Character-level: Split into characters

### Tokenizer Usage

**Load Tokenizer**:
```python
from transformers import AutoTokenizer

model_name = "mistralai/Mistral-7B-v0.1"
tokenizer = AutoTokenizer.from_pretrained(model_name)

# Set padding token (if not set)
if tokenizer.pad_token is None:
    tokenizer.pad_token = tokenizer.eos_token
```

**Tokenize Text**:
```python
text = "Hello, world!"
tokens = tokenizer(text, return_tensors="pt")

print(tokens)
# {
#   'input_ids': tensor([[1, 1059, 29, 1917, 2]]),
#   'attention_mask': tensor([[1, 1, 1, 1, 1]])
# }
```

**Batch Tokenization**:
```python
texts = ["Hello, world!", "How are you?"]
tokens = tokenizer(
    texts,
    padding=True,
    truncation=True,
    max_length=512,
    return_tensors="pt"
)
```

### Tokenization Considerations

**Sequence Length**:
```python
# Check sequence lengths
lengths = [len(tokenizer.encode(text)) for text in texts]
print(f"Average length: {sum(lengths)/len(lengths):.2f}")
print(f"Max length: {max(lengths)}")
print(f"95th percentile: {sorted(lengths)[int(len(lengths)*0.95)]}")
```

**Special Tokens**:
```python
# Add special tokens
tokenizer.add_special_tokens({'pad_token': '[PAD]'})

# Check special tokens
print(f"Pad token: {tokenizer.pad_token}")
print(f"EOS token: {tokenizer.eos_token}")
print(f"BOS token: {tokenizer.bos_token}")
```

## Data Quality Assessment

### Quality Metrics

**1. Diversity Metrics**
```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

# Calculate similarity between samples
vectorizer = TfidfVectorizer()
tfidf = vectorizer.fit_transform(texts)
similarities = cosine_similarity(tfidf)

# Average similarity (lower is more diverse)
avg_similarity = similarities.mean()
print(f"Average similarity: {avg_similarity:.3f}")
```

**2. Coverage Metrics**
```python
# Check coverage of topics/categories
category_counts = df['category'].value_counts()
coverage = len(category_counts) / total_categories
print(f"Category coverage: {coverage:.2%}")
```

**3. Consistency Metrics**
```python
# Check for consistent formatting
format_consistency = df['format'].value_counts(normalize=True)
print(f"Format consistency: {format_consistency.max():.2%}")
```

### Data Quantity Guidelines

**Minimum Data Requirements**:
- **Instruction Tuning**: 1000-10000 examples
- **Domain Adaptation**: 5000-50000 examples
- **Task-Specific**: 100-1000 examples (for simple tasks)
- **Complex Tasks**: 10000+ examples

**Diminishing Returns**:
- Performance plateaus after certain data quantity
- Quality > quantity
- Focus on data curation
